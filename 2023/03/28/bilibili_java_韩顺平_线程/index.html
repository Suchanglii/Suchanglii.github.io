初探线程

当main线程启动一个子线程 Thread-0 ,主线程不会阻塞,会继续执行

注意 :==并不是主线程结束了,整个进程就结束了,不一定==(如果还有其他的线程还没有结束,还是会继续执行)

直接调用的话(==串行化==),就是一个普通方法,没有真正的启动一个线程,就会把run() 执行完,才会继续往下执行,线程会阻塞

Synchronized

线程同步机制

  1. 在多线程编成,一些敏感数据不允许被多个线程同时访问,此时就是用同步访问技术,==保证数据在任何同一时刻,最多由一个线程访问==,以保证数据的完整性
  2. 线程同步,即当有一个线程在对内存进行操作时,其他线程不可以对这个内存地址进行操作,知道该线程完成操作,其他线程才能对该内存地址进行操作

同步的具体方法

  1. 同步代码块

  2. synchronized 还可以放在方法声明中，表示整个方法-为同步方法

互斥锁

局限性: 导致程序的执行效率降低

同步方法 (非静态的) 的锁是加在当前对象的

同步方法 (静态的) 的锁是加在类本身的

释放锁

  1. 当前线程的同步方法,同步代码块执行结束
  2. 当前线程在同步代码快,同步方法中遇到break,return
  3. 当前线程在同步代码快,同步方法中出现了未处理的Error 或者Exception,导致异常结束
  4. 当前线程在同步代码快,同步方法中执行了线程对象的wait() 方法,当前线程暂停,并释放锁

下面操作不会释放锁

  1. 线程执行同步代码块或同步方法时,程序调用Thread.sleep() , Thread.yield()方法,线程暂停当前线程的执行,不会释放锁

  2. 线程执行同步代码块,其他线程调用了该线程的suspend()方法将该线程挂起,该线程不会释放锁

    应该尽量避免使用suspend()与resume()来控制线程,方法不再推荐使用
