Map接口

!!!注意理解,有些方法离不开下面的东西

  1. K-V 左后是HashMap$Node node = newNode(hash,key,value,null)

  2. K-V 为了方便程序员的遍历,还会创建 EntrySet 集合,该集合存放的元素类型 Entry, 而一个Entry 对象就有k,v EntrySet<Entry<K,V>> 即: transient Set<Map.Entry<K,V> entrySet;

    • Entry<K,V>这个东西是Map接口中的一个接口

    • EntrySet 与 Node 一样,都是 HashMap 的内部类,==Entry接口(这是 Map 接口中的接口)定义了数据存储格式==)

      Node直接实现了Entry 接口,而EntrySet 这个集合中存放的元素是实现了Entry 接口的,有一定的联系的!!!

  3. 在EntrySet 中,定义的类型是 Map.Entry ,但是实际上存放的还是HashMap$Node,这是因为HashMap$Node implements Map.Entry<K,V>

  4. 当把HashMap$Node 对象,存放到 entrySet 就方便我们遍历,因为Map.Entry 提供了重要的方法 –> getKey(),getValue()

  5. 其实是把Node转成Entry,然后存放到EntrySet中

  6. Node没有提供相应的获取key,value的方法

TreeSet (与HashSet的最大区别–可以排序)

  1. 当我们使用无参构造器创建 TreeSet 时,仍然是无序的
  2. 希望按照字符串大小来排序,可以使用有参构造器传入一个比较器(匿名内部类),并指定排序规则

注意这句:这会把传进来的对象尝试转换成Comparable 接口对象,如果传入的对象没有实现 Comparable 接口的方法,那么会抛出异常

思考:为什么TreeSet能够实现排序???

因为传递了一个 Comparable 接口,使用这个接口能够自定义排序方式,TreeSet 提供了一系列的有参构造方法供自定义

HashSet: hashCode() +equals(),底层先通过存入对象,进行运算得到一个hash值,通过hash值得到对应的索引,如果发现table索引所在位置,没有数据,就直接存放,如果有数据,就进行equals比较[遍历比较],如果比较后,不相同,就加入,否则就不加入
