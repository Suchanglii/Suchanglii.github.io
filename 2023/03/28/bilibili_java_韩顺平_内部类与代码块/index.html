P.383

main方法刨析

main方法是由java虚拟机调用的

  • 方法只能是public的,因为java虚拟机和main不在同一个包中

  • static :因为java虚拟机在调用main方法的时候不需要创建类实例

  • String[] args: 这个参数是在执行参数的时候传进去的

    ==如何传入args形参==

1. 静态方法main能过直接访问本类的静态成员
2. 静态方法main不可以访问本类的非静态成员
3. 静态方法main 要访问本类的非静态成员,需要先创建对象,再调用即可

P.386

代码块快速入门

1. 当我们在构造器中需要有重复的工作的时候,可以将代码提取到静态代码块中
2. 代码块的调用顺序优先于构造器

P.387

静态代码块

作用: 对类进行初始化

静态代码块随着1. 类的加载而执行,2.并只会执行一次.3.调用静态属性的时候 ,也会被调用
普通代码块 1.每创建一个对象,就执行.2.而且只是使用静态属性,普通代码块不会被执行

类什么时候被加载

  1. 创建对象实例时
  2. 创建子类对象实例,父类也会被加载,而且,父类先被加载,子类后被加载
  3. 使用类的静态成员时

1. static代码块是类加载时,执行.调用一次
2. 普通代码块石创建对象时调用,创建一次,调用一次
3. 类加载的三种情况

P.388

创建对象时,在一个类调用顺序

重点,难点

  1. 同时存在静态代码块和静态属性:两者的优先级是相同的 ,谁在前,谁先执行
  2. 普通代码块一样的
  3. 最后调用构造器

P.389

构造器的最前面其实隐含了super()与调用普通代码块
重点

P.390

重点,难点,面试题

创建一个子类对象时,他们的静态代码块,静态属性初始化,普通代码块,普通属性初始化,构造方法的调用顺序如下:

  1. 父类的静态代码块和静态属性(优先级一样,按照定义顺序执行)
  2. 子类的静态代码块和静态属性(同上)
  3. 父类的普通代码块和普通属性初始化(同上)
  4. 父类的构造方法
  5. 子类的普通代码块和普通属性初始化(同上)
  6. 子类的构造方法

静态代码块只能直接调用静态成员,普通代码块可以调用任意成员

P.395

final关键字

  1. final与static往往搭配使用,效率更高,不会导致类加载
  2. final修饰的而属性是静态的,则初始化位置只能是
    1. 定义时
    2. 在静态代码块中(不能再构造器中赋值)
  3. final修饰的属性再定义时,必须赋初值,并且以后不能更改
    1. 定义时
    2. 构造器中
    3. 代码块中

P.412

内部类

分类:

  1. 定义在外部类局部位置上(比如方法内):
    1. 局部内部类(有类名)
    2. 匿名内部类(没有类名),重点
  2. 定义在外部类的成员位置上:
    1. 成员内部类(没有static修饰)
    2. 静态内部类(使用static修饰)

局部内部类

  1. 定义在外部类的==局部位置==

  2. 可以直接访问外部类的所有成员,包含私有的

  3. 不能添加访问修饰符(==内部类的地位相当于是个局部变量==),但是可以用final修饰(局部变量能用final修饰)

  4. ==作用域==:仅仅在定义它的方法或者代码块中

  5. 局部内部类可以直接访问外部类的成员,比如

  6. 外部类在方法中,可以创建Outer02对象,然后调用方法即可

  7. 外部其他类不能访问局部内部类

  8. 如果外部类与局部内部类的成员重名的时候,遵循就近原则,如果想访问外部类的成员,可以使用(==外部类名.this.成员==)去访问

    外部类名.this.成员的解释:Outer02.this 本质就是外部类的对象,即哪个对象调用了m1,Outer02.this就是哪个对象

匿名内部类的使用细节

  1. 匿名内部类比较奇特,因为匿名内部类既是一个类的定义,同时它本身也是一个对象,因此从语法上看,它既有定义类的特征,也有创建对象的特征.因此可以调用匿名内部类方法

  2. 可以直接访问外部类的所有成员,包括私有的

  3. 不能添加访问修饰符,因为他的低位就是一个局部变量

  4. 作用域:仅仅再定义他的方法或代码块中

  5. 外部其他类不能访问匿名内部类

匿名内部类最佳实践

1.

这种用法的优点:

传统方式如果有需求变动,比如说:sout(“输出的内容变了”);像这种变动会直接影响所有的实例,==使用匿名内部类实现的话,只会改变局部的内容,而不需要从实现类上改动.==

成员内部类

==定义位置==:定义在外部类成员位置上,并且不能有static修饰

  1. 可以访问外部类的所有属性,包括私有的

  2. 可以用修饰符控制访问,因为==定位是一个成员==

  3. 可以调用外部类所有成员

  4. 使用方法就是创建内部类对象,然后调用

  5. 外部其他类—访问—->成员内部类:

    1. 使用外部类对象new一个内部类对象,(相当于把new Inner08()当作是outer08的成员)

      通过外部类实例new内部类

    2. 使用方法new好了传出来

  6. 内部类与外部类成员重名时,遵循就近原则

静态内部类

定义在外部类成员位置,并且有static修饰

特点:

  1. 可以访问外部类的所有静态成员,包含私有的,但不能直接访问非静态成员
  2. 可以添加任意访问修饰符
  3. 作用域为整个类体

内部类使用的原因
1. 封装性
2. 实现多继承
3. 用匿名内部类实现回调功能
4. 解决继承及实现接口出现同名方法的问题
5. 可以有效的将有一定关系的类组织在一起，又可以对外界有所隐藏
6. 方便编写事件驱动程序
7. 方便编写多线程代码
