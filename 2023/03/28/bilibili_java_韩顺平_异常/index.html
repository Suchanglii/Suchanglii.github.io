异常(exception)

==异常处理机制主要处理检查性异常而不是非检查性异常和Error==

异常可以分为两大类:

  1. Error
  2. exception
    1. 编译时异常
    2. 运行时异常

异常体系图

  • 运行时异常,(编译器检查不出来)一般是编成时的逻辑错误,编译器一样不做强制处理,尽量避免即可.

  • 编译时异常是编译器强制要求处理的异常

异常处理的方式

try-catch-finally

throws处理机制

==运行时异常有默认处理方式==,父类方法抛出运行时异常,子类重写后,不必要同样抛出异常,不会报错的
